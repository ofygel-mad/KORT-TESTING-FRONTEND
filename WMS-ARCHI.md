P0 - Бизенс-логика электронной складской системы

P0.1 - Что такое склад? 
Склад - это место, где хранятся данные по остаткам товаров. 


P0.2 - Как формируются остатки товаров? 

1. Остатки товаров формируются по начальным остаткам.
1.1. Начальный остаток формируется один раз при взаимодействии со складом или при его полном сбросе(обнулении). По сути начальный остаток - это то, когда регистрация товаров на складе происходит по фактическому наличию товара на момент заполнения склада. 

P0.3 - Как работает начальный остаток и как его вносить в систему?

Начальный остаток формируется из первичного документа (что-то наподобие таблиц), имеет определённую структуру и шаблон. Тем самым этот документ будет являться источником правды для формирования начальных остатков. 

P0.4 - Что такое Приход? 

Приходная накладная (название документа) - вид накладной, которая помогает зафиксировать пополнение товаров на склад на момент фактического прихода какого-либо товара. 

P0.5 - что такое Расходы? 

Расходом может являться всё что уменьшает остатки на складе. 
Факты расходов могут быть по разным причинам: 
1. бывает товар вынесен из склада и обратно возвращён - треубется документ или система отметки движения этого товара, чтобы предотвратить неизвестное исчезнование (может выявиться недостача в момент инвентаризации)
2. расход может быть по факту срока годности товара для скоропортящихся видов товаров 

P0.4 и P0.5 - движение этих систем формирует "остаток на конец"

понятие "Остаток на конец" в этой ситуации означает запечатление всего факта по расхода и приходам на момент завершения сессии по учёту расхода и прихода. 

P1 - Метод накопления 

Что такое метод накполения относительно нашего склада? 
Логика склада - это когда расчёт прихода и расхода рассчитываются по системе A+B-C=D, где если по примеру А - это 0, B - 10, C - 4, D=6 "0+10-4=6" где 0 - это начальный остаток, 10 - это приход, -4 - это расход (продажа, и любое убытие), а 6 - это Остаток на конец. Тем самым система склада работает так, что формирование любого заказа ставит отметку минус в зависимости от количества товара, которая пойдёт в расход. К примеру, менеджер создаёт заказ на 2 позиций разных товаров, в этой ситуации на складе автоматически появляется -2, затем этот товар после производства в Цехе идёт на склад, где приход на склад этих двух товаров означает +2 и тем самым возникает баланс в виде нуля.

WMS: Метод накопления (Accumulation Method)
1. Определение
Метод накопления — подход к формированию складских остатков, основанный на принципе обратного расчёта. Данные о складе накапливаются постепенно: не путём ввода начальных остатков вручную по всей номенклатуре, а через фиксацию фактических движений товара (расходов и приходов), начиная с нулевого состояния.
________________________________________
2. Базовая формула
Начало + Приход − Расход = Конец
Переменная	Описание
Начало	Начальный остаток на момент первой операции по товару
Приход	Поступление товара (с базара, цеха, аутсорс)
Расход	Выдача по заказу
Конец	Расчётный остаток на конец периода
________________________________________
3. Описание товара (единица номенклатуры)
Каждая позиция склада идентифицируется по составному ключу:
[Модель] — [Размер] — [Цвет] — [Пол] — [Длина]
Пример:
Баян Сулу — 42 — Оранжевый — Женский — Длинный
________________________________________
4. Принцип работы метода
4.1 Триггер — появление расхода (заказа)
1.	В системе поступает заказ на выдачу товара (например, 2 шт.)
2.	Система автоматически создаёт запись в складе для данной позиции номенклатуры, если её ещё не существует
3.	Значение остатка становится отрицательным (0 + 0 − 2 = −2)
4.	Система генерирует алерт — сигнал о необходимости фактической сверки остатка
4.2 Сверка фактического остатка
После получения алерта сотрудник физически пересчитывает товар на складе и вносит фактическое значение как начальный остаток.
Пример:
Факт на складе: 5 шт.
→ Начало = 5
→ Приход = 0 (прихода не было)
→ Расход = 2 (заказ)
→ Конец = 5 + 0 − 2 = 3
4.3 Приход товара (с цеха / с базара)
При поступлении товара значение вносится в колонку Приход. Система пересчитывает остаток с учётом нового прихода и уже имеющегося начального остатка.
Пример — ручной приход:
Приход: 10 шт.
Факт после прихода: 11 шт.
→ Начало = 1 (вычислено обратным расчётом: 11 − 10)
→ Приход = 10
→ Расход = 2 (заказы)
→ Конец = 1 + 10 − 2 = 9

6. Алерт (сигнал сверки)
•	Отображается рядом с записью товара при невалидном (отрицательном или нулевом начальном) остатке
•	Сигнализирует: «Необходимо физически пересчитать остаток»
•	Снимается после внесения подтверждённого начального значения
•	Применяется как при расходе, так и при ручном приходе «с цеха и с базара»

8. Ключевые принципы
1.	Нет необходимости вводить весь склад вручную — номенклатура появляется по мере возникновения операций
2.	Расход (заказ) — первичный триггер создания записи о товаре
3.	Алерт — обязательный этап между автоматическим созданием и достоверным остатком
4.	Приход уточняет историю — позволяет вычислить начальный остаток обратным счётом
5.	Система накапливает данные постепенно — со временем картина склада становится полной
