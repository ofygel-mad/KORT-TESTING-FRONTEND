! Жалобы и корневые причины

БАГ 1 Карточки — слишком большие скругления, выглядят как пилюли

Строки таблицы имеют border-radius: 22px — они выглядят как огромные плавающие карточки, а не как строки таблицы. Особенно заметно когда строк много: список похож на набор отдельных блоков.

ChapanProduction.module.css строка ~176
@@ .card @@
border: 1px solid ...;
- border-radius: 22px;
+ border-radius: 10px;
background: ...;
БАГ 2 Кнопка «Готово» — полная таблетка, обрезается overflow

Кнопка имеет border-radius: 999px (pill) внутри карточки с overflow: hidden. Комбинация приводит к тому что углы кнопки обрезаются краями карточки. Визуально кнопка «торчит» и режется справа — особенно при узком вьюпорте.

ChapanProduction.module.css строка ~253
@@ .doneBtn @@
min-height: 42px;
- border-radius: 999px;
+ border-radius: 10px;
border: 1px solid ...;
БАГ 3 Хедер таблицы обрезан справа — «ДЕЙСТВИЕ» не видно

Это архитектурный баг: .tableHeader и .cardList находятся в разных DOM-контейнерах. При ширине <1080px у .cardList включается overflow-x: auto с min-width: 1030px на карточках, а хедер остаётся в родительском .root без скролла — его правая часть просто обрезается вьюпортом и колонки расходятся.

⚠️ Положить .tableHeader и .cardList в один общий <div> с overflow-x: auto — они должны скроллиться горизонтально как одно целое.
ChapanProduction.tsx — структурное изменение JSX
@@ return ( ... ) @@
<header className={styles.pageHeader}>...</header>
+ <div className={styles.tableScrollWrap}>
<div className={styles.tableHeader}>...</div>
<div className={styles.cardList}>...</div>
+ </div>
ChapanProduction.module.css — новый класс
+ .tableScrollWrap {
+ overflow-x: auto;
+ -webkit-overflow-scrolling: touch;
+ }
ChapanProduction.module.css — убрать overflow из cardList
@@ @media (max-width: 1080px) @@
.cardList {
- overflow-x: auto;
- padding-bottom: 18px;
}
БАГ 4 Большой gap между строками — не таблица, а стопка карточек

gap: 12px в .cardList создаёт слишком большой вертикальный отступ между строками — они "плавают" как отдельные карточки. Для таблицы производственных задач нужен более компактный ритм.

ChapanProduction.module.css строка ~225
@@ .cardList @@
display: flex;
flex-direction: column;
- gap: 12px;
+ gap: 4px;
padding: 18px 24px 28px 20px;
БАГ 5 Хедер таблицы: border-top + inset shadow дают визуальный «отрыв»

У .tableHeader есть border-top белый полупрозрачный и box-shadow: inset 0 1px 0 — вместе они дают иллюзию что хедер «висит» над страницей и не примыкает к pageHeader снизу.

ChapanProduction.module.css строка ~108
@@ .tableHeader @@
background: linear-gradient(...);
- border-top: 1px solid color-mix(in srgb, white 18%, transparent);
border-bottom: 1px solid ...;
- box-shadow: inset 0 1px 0 color-mix(in srgb, white 16%, transparent);

⚠️ Важно: position: sticky на .tableHeader не работает внутри overflow-x: auto контейнера (браузерное ограничение CSS). Поэтому position: sticky у .tableHeader нужно убрать или оставить только как вертикальный sticky через JS-синхронизацию скролла если это критично для UX.
