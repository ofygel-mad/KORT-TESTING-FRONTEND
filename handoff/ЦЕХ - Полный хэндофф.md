## 7. JS — логика сортировки (`sortTasks`)

Вызывается внутри `useMemo`. Входной массив не мутируется — возвращается новый `[...arr].sort(...)`.

```
Приоритет сортировки:
  1. Просроченные (due < сегодня) — идут ПЕРВЫМИ
  2. Среди просроченных — по due по возрастанию
  3. Среди не просроченных — по due по возрастанию
  4. При одинаковой due — urgent:true выше
  5. Остальное — исходный порядок
```

```js
function sortTasks(arr) {
  const today = new Date();
  today.setHours(0, 0, 0, 0);

  return [...arr].sort((a, b) => {
    const aD = a.order.due ? new Date(a.order.due) : new Date('9999-12-31');
    const bD = b.order.due ? new Date(b.order.due) : new Date('9999-12-31');
    aD.setHours(0,0,0,0); bD.setHours(0,0,0,0);

    const aOver = aD < today;
    const bOver = bD < today;

    if (aOver && bOver) {
      if (aD.getTime() !== bD.getTime()) return aD - bD;
      return a.order.urgent ? -1 : 1;
    }
    if (aOver !== bOver) return aOver ? -1 : 1;
    if (aD.getTime() !== bD.getTime()) return aD - bD;
    if (a.order.urgent !== b.order.urgent) return a.order.urgent ? -1 : 1;
    return 0;
  });
}
```

## 8. JS — цепочка фильтрации + сортировки (`useMemo`)

```js
const tasks = useMemo(() => {
  let r = ALL_TASKS.filter(t => !done.has(t.id));   // 1. убрать завершённые

  if (search) {                                       // 2. поиск по № заказа и названию
    const q = search.toLowerCase();
    r = r.filter(t =>
      t.order.num.toLowerCase().includes(q) ||
      t.product.toLowerCase().includes(q)
    );
  }

  if (dueFilter)  r = r.filter(t => t.order.due  === dueFilter);  // 3. фильтр по сроку
  if (recvFilter) r = r.filter(t => t.order.recv === recvFilter); // 4. фильтр по приёму

  if (hideUnsel && selected.size > 0)                             // 5. только выбранные
    r = r.filter(t => selected.has(t.id));

  const sorted = sortTasks(r);                                    // 6. сортировка

  // 7. добавить порядковый номер позиции внутри каждого заказа
  const withPos = [];
  const orderPos = new Map();
  for (const t of sorted) {
    const pos = (orderPos.get(t.order.num) || 0) + 1;
    orderPos.set(t.order.num, pos);
    withPos.push({ ...t, position: pos });
  }
  return withPos;
}, [search, dueFilter, recvFilter, selected, hideUnsel, done]);
```

## 9. JS — действия пользователя

### Выбор позиции (чекбокс)
```js
const toggleSel = id => {
  const n = new Set(selected);
  n.has(id) ? n.delete(id) : n.add(id);
  setSelected(n);
};
// → строка получает класс .is-sel (синий фон)
// → при selected.size > 0 появляется .sel-bar и кнопка "Только выбранные"
```

### Кнопка «Готово»
```js
const markDone = id => {
  setDone(p => { const n = new Set(p); n.add(id); return n; });
  setSelected(p => { const n = new Set(p); n.delete(id); return n; });
  // → позиция уходит из useMemo (done.has(id) === true)
  // → карточка пропадает из списка
  // ПРОДАКШН: здесь должен быть POST /api/tasks/:id/complete
};
```

### Фильтр «Только выбранные»
```js
// Кнопка в toolbar:
<button onClick={() => setHideUnsel(v => !v)}>
  {hideUnsel ? 'Все' : 'Только выбранные'}
</button>
// → при hideUnsel=true useMemo оставляет только selected задачи
// → кнопка получает класс .active (синяя обводка)
```

### Снять все выборки
```js
// Клик по чекбоксу в .sel-bar:
onClick={() => setSelected(new Set())}
// → selected очищается → .sel-bar скрывается → hideUnsel автоматически теряет смысл
```

## 10. Компонент `TaskRow` — нюансы сетки

Строка использует **`display:grid`** с `grid-template-rows: 42px auto`.  
Когда у позиции есть `task.notes`, добавляется вторая строка сетки.

Ячейки, которые НЕ должны разрываться на 2 строки, получают `className="cell span-2"` → `grid-row: 1 / 3`.  
Это: чекбокс (col-1), badge (col-2), номер (col-3), кол-во (col-8), размер (col-9), даты (col-10, col-11), кнопка (col-12).

`.cell-note` занимает `grid-column: 4 / 8` и `grid-row: 2` — т.е. под колонками "Товар", "Пол", "Длина", "Цвет".

## 11. Что нужно для продакшн-интеграции

| Задача | Что менять в коде |
|---|---|
| Загрузить заказы из БД | Заменить `ALL_TASKS` на `useEffect` + `fetch('/api/tasks?stage=production')` |

| Авторизация швеи | Добавить `assignee_id` в структуру task, фильтровать по текущему пользователю |

| Реалтайм (новые заказы без перезагрузки) | WebSocket или polling `setInterval` |
| Этап после Цеха | `stage: 'quality_control'` — задача уходит в раздел Контроля качества |
